### Завдання: "Управління інтернет-магазином"

#### Опис:
Розробіть систему управління інтернет-магазином, яка дозволить:
- Управляти товарами (додавання, редагування, видалення).
- Управляти замовленнями (оформлення замовлення, оновлення статусу, додавання товарів до замовлення).
- Керувати клієнтами (реєстрація клієнтів, перегляд інформації, історія замовлень).
- Відстежувати склади та кількість товарів (оновлення кількості на складах після продажу).
- Виводити звіти (наприклад, найпопулярніші товари, загальна сума замовлень за певний період).

Дані зберігаються у форматі JSON для забезпечення можливості збереження стану системи і її відновлення при перезапуску програми.

#### Сутності:
1. Товар:
   - ID (унікальний ідентифікатор).
   - Назва.
   - Категорія.
   - Ціна.
   - Кількість на складі.
   - Опис.

2. Замовлення:
   - Номер замовлення.
   - Список товарів (кожен товар має кількість і ціну).
   - Клієнт, який зробив замовлення.
   - Загальна сума.
   - Статус (наприклад, "Очікує", "Відправлено", "Завершено").

3. Клієнт:
   - ID (унікальний ідентифікатор).
   - Ім'я.
   - Електронна пошта.
   - Телефон.
   - Адреса.
   - Історія замовлень.

4. Склад:
   - ID складу.
   - Назва складу.
   - Місткість.
   - Товари (товари і їх кількість на складі).

#### Операції:
1. Управління товарами:
   - Додавання нового товару (заповнення інформації про товар).
   - Оновлення інформації про товар (редагування ціни, кількості на складі тощо).
   - Видалення товару.
   - Пошук товарів за різними параметрами (назва, категорія, ціна).

2. Управління замовленнями:
   - Оформлення нового замовлення (вибір товарів і кількості).
   - Оновлення статусу замовлення.
   - Додавання або видалення товарів із замовлення.
   - Розрахунок загальної вартості замовлення.
   - Збереження замовлень в JSON для подальшої обробки.

3. Клієнти:
   - Реєстрація нових клієнтів.
   - Перегляд інформації про клієнта.
   - Перегляд історії замовлень клієнта.
   - Видалення або редагування інформації про клієнта.

4. Склад:
   - Додавання нових складів.
   - Оновлення кількості товарів на складі після оформлення замовлення.
   - Виведення інформації про запаси товарів.

5. Звіти:
   - Виведення списку найпопулярніших товарів за певний період.
   - Підрахунок загальної суми замовлень за день, тиждень або місяць.
   - Виведення залишків на складі.

#### Структура модулів:
1. module_products.py:
   - Клас Product, який представляє товар.
   - Функції для додавання, редагування, видалення товарів.
   - Функція для збереження та завантаження даних про товари з JSON файлу.

2. module_orders.py:
   - Клас Order, який представляє замовлення.
   - Функції для створення, оновлення статусу, розрахунку загальної вартості замовлення.
   - Функція для збереження та завантаження замовлень з JSON файлу.

3. module_clients.py:
   - Клас Client, який представляє клієнта.
   - Функції для реєстрації клієнтів, редагування їх інформації, перегляду історії замовлень.
   - Функція для збереження та завантаження даних про клієнтів з JSON файлу.

4. module_warehouses.py:
   - Клас Warehouse, який представляє склад.
   - Функції для додавання складів, оновлення запасів товарів.
   - Функція для збереження та завантаження даних про склади з JSON файлу.

5. module_reports.py:
   - Функції для генерації звітів про найпопулярніші товари та загальну кількість замовлень.

6. main.py:
   - Головна програма, яка дозволяє взаємодіяти з користувачем, управляти товарами, замовленнями, клієнтами, складами і генерувати звіти.


#### Додаткові вимоги:
- Клієнт може мати кілька замовлень.
- Склад оновлює кількість товарів після кожного замовлення.
- Звіти повинні враховувати тільки завершені замовлення.
